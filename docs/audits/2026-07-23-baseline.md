# Audit de référence avant remédiation

Date : 2026-07-23  
Branche de départ : `main`

Ce document fige les mesures observées avant l’implémentation du chantier SEO,
performance, responsive, accessibilité, sécurité et fiabilité.

| Contrôle | Résultat initial |
| --- | --- |
| ESLint | vert |
| Tests unitaires | 46 tests verts |
| Build Next.js | vert |
| Playwright | 70 réussis, 12 ignorés, 6 en échec |
| Audit production | 15 vulnérabilités `high`, 16 `moderate`, 4 `low` |
| JavaScript initial public | environ 267–285 Kio Brotli |
| CSS global brut | environ 196,5 Kio |
| Dossier `public/` | environ 99 Mio |

Les six régressions E2E de référence concernaient le rayon du rail desktop, le
résumé de recherche assistée Techno, la visibilité du déclencheur
d’authentification à 1440 px, le CTA des recherches sauvegardées, la couleur de
note des cartes sensation et la hauteur du manifeste mobile.

Les mesures Lighthouse de référence doivent être produites sur une Preview
Vercel, car les données Harvest et les conditions réseau ne sont pas
reproductibles de façon fiable pendant un build local. Le workflow CI ajouté
au projet conserve les rapports uniquement en cas d’échec.
